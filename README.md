# TCP-Sliding-Window

## Overview
This project implements a basic TCP Sliding Window protocol simulation for CS 258. It consists of a client and server that communicate over a network, simulating packet transmission with probabilistic drops, retransmissions, and performance tracking. The client sends a large number of packets (default: 10 million) using a sliding window, while the server receives and acknowledges them. Data is logged to CSV files for analysis, and graphs can be generated to visualize window sizes, received sequences, and packet drops.

## Features
- Sliding window protocol with configurable window size (default: 1024)
- Probabilistic packet dropping (default: 1%) to simulate network unreliability
- Retransmission logic triggered by timeouts or duplicate ACKs
- Cumulative ACKs from server
- Goodput calculation (original packets sent / total attempts)
- CSV logging for performance metrics
- Graph generation using matplotlib

## Prerequisites
- Python 3.x
- matplotlib library (`pip install matplotlib`)

## Installation
1. Clone or download the repository.
2. Install dependencies: `pip install matplotlib`
3. Ensure the client and server can communicate (same network or adjust SERVER_IP in client.py).

## Usage
1. **Start the Server**: Run `python server.py` on the server machine. It will listen on port 5050.
2. **Start the Client**: Run `python client.py` on the client machine. Update SERVER_IP in client.py to the server's IP address.
3. **Generate Graphs**: After the transfer completes, run `python generate_graphs.py` to create PNG graphs from the CSV files.

## Configuration
- **Client Parameters** (in client.py):
  - SERVER_IP: IP address of the server
  - WINDOW_SIZE: Sliding window size
  - DROP_PROBABILITY: Probability of dropping a packet (0.01 = 1%)
  - TOTAL_PACKETS: Number of packets to send
  - VERBOSE: Enable detailed logging
- **Server Parameters** (in server.py):
  - RECEIVER_WINDOW_SIZE: Receiver window size
  - VERBOSE: Enable detailed logging

## Output
- **CSV Files**:
  - sender_window.csv: Sender window size over time
  - receiver_window.csv: Receiver window size over time
  - received_seq.csv: Sequence numbers received
  - drops.csv: Dropped packet sequences
  - retransmissions.csv: Retransmission counts
- **Graphs** (generated by generate_graphs.py):
  - window_sizes.png: Window sizes over time
  - received_seq.png: Received sequences
  - drops.png: Dropped sequences scatter plot
- **Console Output**: Progress updates, summaries, and goodput metrics
